# WinBattery 电脑电池管家 PRD
## 文档基本信息
**产品名称**：WinBattery 笔记本电池监控工具
**产品定位**：Windows 平台轻量纯净电池检测软件，对标安卓 AccuBattery，无广告、无捆绑、无后台、全本地运行，新增**中英双语国际化、多套UI主题、系统主题自动跟随**能力。
**目标用户**：Windows 笔记本用户、关注电池健康、海外用户、喜欢自定义界面风格、讨厌广告工具软件用户
**运行平台**：Windows 10 1909+ / Windows 11
**开发难度总结（前置）**

# 1. 产品概述
## 1.1 产品简介
一款专注笔记本电池信息查看、实时监控、损耗计算、充放电曲线、耗电分析、电池养护的轻量桌面软件。
全程**仅读取系统原生WMI电源接口数据**，不修改硬件、不修改系统电源策略、不上传任何用户数据、无联网、无广告、无捆绑。
本次迭代新增核心特性：
1. **全软件中英双语切换**，界面所有文本、提示、注释、说明、建议全部双语
2. **多主题皮肤系统**，支持亮色/暗色/自定义主题
3. **主题默认跟随Windows系统全局深色浅色模式**，一键切换、自动同步系统

## 1.2 产品优势
1. 体积极小，单文件exe免安装，内存占用低
2. 数据完全来自Windows原生接口，权威准确
3. 全本地存储，无网络请求，隐私安全
4. 国际化双语适配，支持国内/海外用户使用
5. 主题自适应系统，兼顾美观与使用习惯
6. 仅专注电池模块，不附加多余硬件检测功能

## 1.3 运行约束
- 仅支持笔记本内置电池，台式机无内置电池直接提示
- 无需管理员权限即可读取全部基础电池数据
- 支持Windows系统深浅色模式原生同步

# 6. 数据与技术实现方案（完整版）
## 6.1 底层电池数据读取（完全不变）
依旧基于 **WMI Windows系统管理接口** 原生读取，无驱动、无破解、无权限风险
核心WMI类：`Win32_Battery`、`Win32_PowerMeter`
可获取全部字段：设计容量、满充容量、循环次数、电压、温度、厂商、充放电状态等
备用兜底方案：调用系统 `powercfg /batteryreport` 命令解析报告

### 核心计算公式不变
\[
健康度 = \frac{当前满充容量}{出厂设计容量} \times 100\%
\]
\[
损耗 = 100\% - 健康度
\]

## 6.2 新增：国际化多语言技术方案
### 架构方案
**独立语言资源包架构**
1. 单独建立语言字典文件（json/xml），中文、英文两套键值对
2. 程序内所有文本统一通过Key索引读取，不硬编码任何界面文字
3. 全局语言管理器，切换语言时全局重载资源包，刷新全部UI文本
4. 首次启动自动检测 `GetUserDefaultUILanguage` Windows系统原生API，获取系统语言初始化

### 适配语言API
Windows原生系统语言读取API，精准匹配系统界面语言。

## 6.3 新增：主题系统技术方案
### 1）系统主题跟随原理
调用Windows原生API读取系统深浅色模式注册表项：
```
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Themes\Personalize\AppsUseLightTheme
```
- 值 = `1` → 系统浅色模式
- 值 = `0` → 系统深色模式

软件监听该注册表变更，实现**系统改主题，软件自动同步**。

### 2）主题样式架构
采用 **CSS/样式表分离架构**（Python/C#通用）
1. 把所有颜色、背景、字体色、边框色全部抽离为主题变量
2. 三套主题各自维护一套配色变量集
3. 切换主题仅切换变量集，全局UI自动渲染
4. 图表库颜色绑定主题变量，自动适配对比度

## 6.4 推荐开发技术栈（适配双语+主题最优）
### 方案A（新手首选 Python + CustomTkinter）
- 界面：CustomTkinter（原生支持深浅主题、自适应、现代UI，天生适配系统跟随）
- 数据读取：`wmi` 库
- 图表：`matplotlib` / `ctk_charts`
- 多语言：独立json多语言字典
- 系统主题监听：注册表监听库
- 配置持久化：本地json配置文件
**优势**：主题原生支持、代码简单、双语极易实现、UI美观接近手机APP

### 方案B（原生性能最强 C# WinForms/WPF）
- 原生Win32 API读取系统语言、系统主题
- 资源文件.resx做多语言国际化（微软标准国际化方案）
- 样式字典实现主题切换
- 单文件exe无依赖

## 6.5 配置持久化设计
本地单个 `config.json` 文件存储全部用户偏好：
```json
{
  "language": "zh/en",
  "theme_mode": "system/light/dark",
  "auto_start": true,
  "float_window": true,
  "refresh_interval": 5
}
```
程序启动优先读取该配置，无配置文件则采用默认规则：**跟随系统语言 + 跟随系统主题**。
